# Dead Code Cleaner

このスキルは、プロジェクトの健全性を維持するために、参照されていない「デッドコード」を安全に特定し、削除するための手順を提供します。

## ワークフロー

1.  **スキャン**: 参照されていない可能性のあるコード（未使用のファイル、エクスポート、CSS）を特定します。
2.  **安全性チェック**: [deletion-safety.md](references/deletion-safety.md) に基づき、削除しても本当に安全かを確認します。
3.  **バックアップ/確認**: 削除前に影響範囲を確認し、必要に応じてユーザーに確認を求めます。
4.  **クリーンアップ**: コードを削除し、関連するインポート文や設定を整理します。

## タスク別の手順

### 1. 未使用ファイルの特定
インポートされていない `.ts`, `.tsx`, `.css` ファイルを探します。
- `grep_search` を使用して、ファイル名（拡張子なし）が他のファイルで参照されているか確認します。
- エントリポイント（`main.ts`, `page.tsx` など）から辿れないファイルは削除候補です。

### 2. 未使用のエクスポート（関数、型、DTO）
`export` されているが、他のファイルから `import` されていない要素を特定します。
- `export const MyVar` や `export class MyDto` などの定義を検索し、その名前がプロジェクト内でどのように使われているか調査します。

### 3. 未使用の CSS クラス
`globals.css` や CSS Modules で定義されているが、JSX/TSX 内で参照されていないクラスを特定します。
- クラス名（例: `.my-unused-class`）で検索し、使用箇所がなければ削除します。

### 4. 未使用の依存関係
`package.json` に記載されているが、コード内で一度もインポートされていないライブラリを確認します。

## 安全な削除のための注意点
削除を行う前に、必ず [deletion-safety.md](references/deletion-safety.md) を読み、動的な参照（文字列によるアクセスなど）がないか確認してください。
